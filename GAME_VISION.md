# 宝珠游戏目标与实现构想

> 本文档记录游戏希望带给玩家的核心体验、文化表达目标和实现这些目标的设计构想。
> 它用于判断新功能是否值得加入，也用于避免游戏在开发过程中变成普通的合成游戏。

## 一、游戏的核心目标

游戏有两个同等重要的目标：

1. **传播宝珠与酿造文化**
2. **做成一个真正好玩、愿意继续玩的游戏**

文化不能只存在于图鉴和说明文字里，好玩也不能只依赖奖励、数值和关卡难度。理想状态是：玩家因为操作、观察和完成一份饮品，自然理解其中的食材、工艺与情感。

### 核心体验陈述

玩家完成开场和第一章后，应该产生这样的感受：

> 我没有被催促，也没有被惩罚。我亲手把普通食材变成了一份有来处的味道，并让一个具体的地方亮了起来。

玩家应该知道：

- 这是一个温暖、放松、没有激烈竞争的游戏。
- 游戏不会因为试错而羞辱或惩罚玩家。
- 不同食材与工艺有自己的性格和结果。
- 宝珠的味道来自奶、酒酿、花香、时间与手艺的共同作用。
- 制作食物不是为了向系统提交答案，而是在回应一个人的需要。

## 二、“慢”的正确含义

“慢”是游戏的气质，不等于让玩家长时间等待。

### 游戏应该慢在哪里

- 镜头、光线、环境和音乐的变化自然，不突然切断。
- 动画有呼吸感，物品出现、合成和离场都有完整动作。
- 玩家不会被倒计时、排行榜或失败惩罚催促。
- 文案克制，不连续弹出大量说明。
- 玩家可以停下来观察环境、物品和已经完成的成果。

### 游戏不应该慢在哪里

- 第一章不设置长时间发酵等待。
- 新玩家完成第一次合成前不能经历冗长叙事。
- 已经理解的动画和步骤不能反复拖延玩家。
- 不用等待代替玩法，也不用长对白代替故事。
- 不强迫玩家为了获得材料而空等。

### 第一章节奏原则

第一章承担的是“让玩家愿意继续玩”，因此应该快速、连续、低门槛：

- 开场开始后尽快给玩家第一次可操作行为。
- 第一次合成应在几十秒内发生。
- 前三次成功之间保持连续反馈，不插入长弹窗。
- 发酵在第一章主要通过视觉、声音和快速变化表达。
- 真正需要时间管理或较长等待的工艺，在后续章节逐步加入。
- 后续等待期间必须有观察、整理、交换、查看配方等可做内容。

## 三、文化传播方法

### 文化要通过行为被理解

不要只告诉玩家“酒酿需要时间”，而应该让玩家看到原料状态、气泡、香气、色泽和声音发生变化。

不要只在图鉴中写花材产地，而可以通过换物人、环境、器皿和完成后的生活场景体现来源。

不要把现实工艺和游戏化配方混为一谈：

- 游戏配方负责建立易懂、好玩的规则。
- 真实工艺用简短、准确的内容补充。
- 如果游戏规则经过简化，应明确它是“游戏中的酿造语言”。

### 第一章适合传达的文化认识

- 发酵是一种转化，不是一枚普通的合成按钮。
- 不同谷物经过相似工艺会产生不同结果。
- 奶酪与雪酪代表厚实和轻盈两种质地。
- 酒酿可以承载奶香与花香。
- 桂花、玫瑰等花材不是装饰，而是味道与地域记忆的一部分。
- 食物的意义不仅是配方，也包括它被制作给谁、在什么时刻被需要。

## 四、世界与使命

### 用“灯火巷”替代门

当前的门负责展示目标、接收物品、提供提示和判断通关，但它容易让玩家觉得自己在向系统提交答案。

推荐的新表达是一条逐渐亮起的巷子：

- 屏幕上方是一户尚未亮灯的人家、小店或窗台。
- 窗前的托盘展示今天要完成的饮品。
- 合成目标后，成品先完整出现，再落到托盘上。
- 成品化成一缕温暖的光，沿巷子进入窗口。
- 窗户亮起，并出现很短的生活声音、剪影或动作。
- 多目标不再填充门，而是在桌上逐渐摆齐一席饮品。
- 第一章结束时，所有灯火连接起来，形成宝珠的标志或标题。

“村庄”可以作为世界框架，但视觉和故事不能成为泛化的古风村落。场景应吸收田子坊、小巷、小店、酿造器皿和真实生活空间的特征，让文化来源具体可信。

### 玩家身份

玩家不是救世主，也不是参加考试的人。

玩家可以是：

- 初来巷子的酿造学徒。
- 帮小店恢复旧配方的人。
- 为不同住户准备一份饮品的人。
- 在学习传统之后，最终能够创造自己味道的人。

长期成长目标可以是“获得创造自己饮品的自由”，但第一章只需要建立“我能用手艺让这里变温暖”的动机。

## 五、第一章体验结构

第一章应保持快速，同时形成从“会操作”到“理解宝珠味道”的完整弧线。

### 第一缕甜

- 快速教会拖拽与合成。
- 完成甜牛奶，点亮第一户窗口。
- 不显示评分，不打断玩家庆祝。

### 奶酿成酪

- 认识发酵和质地变化。
- 第一章中的过程保持短促，用动画和声音表达转化。
- 真实工艺与游戏规则分开说明，避免错误知识。

### 四种米，四种酿

- 让玩家发现同一种工艺面对不同原料会得到不同结果。
- 强调观察与发现，不把不同结果判为错误。

### 厚与轻

- 奶酪和雪酪代表两种互补质地。
- 配方书提供方向，但最后一步应让玩家自己推导。

### 以酪换花

- 把冷冰冰的交易台改造成巷中的换物摊、花篮或具体人物。
- 交换表达邻里关系，而不只是资源转换。

### 灯火成席

- 综合第一章学到的规则制作经典饮品。
- 不称为 Boss 或测试。
- 完成后看到前面帮助过的人共同出现，形成一席或一条亮起的巷子。

## 六、好玩的来源

### 推导，而不是照做

理想过程是：

1. 玩家短暂困惑。
2. 观察物品、环境或部分配方。
3. 自己推导组合。
4. 验证成功。

操作方法必须明确引导，配方答案则尽量通过线索让玩家自己发现。

### 试错没有挫败

- 合成错误不损失关键资源。
- 错误物品轻轻弹开或形成有趣的旁支产物。
- 不使用刺耳警报、红色大叉或嘲讽文案。
- 连续多次偏离目标后才提供逐步增强的提示。
- 提示说明方向，不直接替玩家完成推导。

### 变化与惊喜

- 相同工艺遇到不同原料产生可理解的变化。
- 支线配方能带来图鉴、环境反应或住户反馈。
- 连续成功让环境更温暖，但不做竞技连击计分。
- 后续章节逐渐加入容器、模具、温度和制作动作，扩展自由度。

### 掌控感

- 玩家知道当前在为谁制作、需要什么、自己已经做到了哪一步。
- 配方书、环境线索和目标展示互相一致。
- 玩家可以选择暂时离开，游戏用温和方式结束当前体验。

## 七、反馈与奖励

### 主要奖励

主要奖励应该是世界发生变化：

- 一扇窗亮起。
- 一户人家的声音或动作出现。
- 巷子增加新的植物、蒸汽、招牌或灯光。
- 已完成的饮品留在桌面或回忆中。

### 次要奖励

- 钻石、任务奖励和图鉴收集可以保留。
- 货币奖励不能抢在情绪反馈之前。
- 推荐顺序：成品完成 → 温暖传递 → 场景亮起 → 安静显示奖励。
- 第一章不突出效率、速度评级和最佳次数。

## 八、应该加入、改变和删除的内容

### 加入

- 每户一个简短、具体的生活情境。
- 一条永久记录玩家成果的灯火巷。
- 合成关键物品时的一句真实文化信息。
- 后续章节等待期间可进行的整理、观察、交换和查阅活动。
- 温和且逐步增强的提示系统。
- 玩家离开时的短句，例如“今天先酿到这里”。

### 改变

- “目标”改成“今日所作”“这一席”或具体需求。
- “通过”改成“灯亮了”“这一份已经送到”等结果表达。
- “交易台”改成符合场景的换物摊、花篮或人物互动。
- 钻石从主要庆祝降为次要奖励。
- 开场故事缩短，先让玩家操作，再逐步展开背景。
- 关间转场强调世界连续变化，减少反复进入和退出界面。

### 删除或避免

- 门作为角色、导师和审判者。
- “Boss”“测试”“最后一关，做出来就通过”等考试式语言。
- 第一章的速度等级、排行榜、倒计时和失败重来。
- 泛化的拯救世界叙事。
- 为了显得“慢”而加入的无内容等待。
- 重复且抢戏的通关弹窗、文字和奖励动画。

## 九、开场设计原则

- 最初的白点可以变成远处第一盏灯，而不是打开一道门。
- 玩家尽快获得糯米与酿造，并完成第一次合成。
- 第一份酒酿化为光，进入远处尚未亮起的窗口。
- 窗户亮起后再出现极短的背景信息。
- 不在开场一次讲完碎片、宇宙、终极配方等全部设定。
- 开场结束时，玩家已经理解“制作一份味道，可以让一个地方变暖”。

## 十、验证标准

每次试玩后应确认玩家能回答：

1. 你在游戏里扮演谁？
2. 你为什么要制作这些饮品？
3. 第一章让你记住了什么食材或工艺？
4. 你是否感到被催促、惩罚或考试？
5. 哪一刻让你最想继续玩？

第一章方向成立的标志：

- 玩家不需要解释就知道如何完成第一次合成。
- 玩家能说出自己是在为巷子里的人制作饮品、传递温暖。
- 玩家至少记住一个与酒酿、谷物、奶酪质地或花材有关的认识。
- 玩家认为节奏流畅，而不是漫长。
- 玩家愿意主动进入下一章，而不是只为了领取奖励。

## 十一、实施顺序建议

1. 先制作第一关的“暗窗 → 托盘 → 成品化光 → 窗户亮起”完整原型。
2. 验证玩家是否理解新的使命，以及动画是否足够清晰。
3. 把现有门逻辑抽象成通用的“目标接收者”，避免一次性重写全部流程。
4. 替换开场中的门，并压缩开场叙事。
5. 将第一章六个目标改写为同一条巷子里的连续事件。
6. 调整通关文案、钻石顺序和结算界面。
7. 最后再删除不再使用的门样式、对白和旧数据。

## 十二、已确认的设计决定

- 游戏的核心目标是“传播文化”和“好玩”，两者不能互相牺牲。
- 游戏应温暖、放松、无激烈竞争，也不通过失败制造挫败。
- 第一章必须保持较快节奏，不能用长等待表达发酵文化。
- “慢”主要通过氛围、动作、声音和无压力结构表达。
- 门应被更温暖、更具体、更有文化来源的目标接收方式取代。
- 推荐使用逐渐亮起的巷子、人家和小店来承载“传递温暖”的结果。
- 具体实现先做第一关原型，验证后再扩展到整个开场和第一章。

## 十三、玩法层面的四种核心感受

目标感、成就感、自由和放松不应该由四套互不相关的系统分别提供。核心玩法循环应同时支撑这四种感受：

> 看见一种生活需要 → 自由选择制作路线 → 通过短而有触感的操作完成饮品 → 送出 → 看见永久变化 → 获得新的制作自由。

| 希望产生的感受 | 对应玩法 | 玩家感受到它的原因 |
|---|---|---|
| 目标感 | 一个清楚的生活需要、可见的制作进度 | 知道现在为谁制作、为什么制作 |
| 成就感 | 自己推导配方、世界永久变化 | 成功来自自己的理解，并留下了结果 |
| 自由 | 多个正确答案、路线选择、自创配方 | 不再只是执行系统规定的唯一答案 |
| 放松 | 无损试错、无倒计时、可随时停下 | 玩家敢于尝试，不担心浪费和失败 |

## 十四、从指定物品到味道愿望

第一章前半段仍然使用明确目标，保证新玩家快速理解：

- 制作甜牛奶。
- 制作奶酪。
- 制作酒酿。

第一章后半段开始加入简单的味道需求，后续章节逐渐替代唯一指定成品：

- 想要一份轻盈、带甜味的酪。
- 想要花香明显，但不要太厚重。
- 今晚有点冷，想喝温暖醇厚的东西。
- 想要适合几个人一起分享的饮品。

### 味道属性

每种饮品可以拥有一组容易理解的味道属性：

- 温暖 / 冰凉
- 厚实 / 轻盈
- 甜润 / 清爽
- 奶香 / 酒酿 / 花香
- 柔和 / 浓烈

只要成品满足主要需要，就是正确答案。不同答案可以产生不同人物反应、文化记录和环境变化，但不设置“低分答案”。

这个系统让玩家从寻找标准答案逐渐转向表达自己的理解，是自由感最重要的来源。

## 十五、永久保留玩家造成的变化

完成一份饮品以后，主要结果不能只是“通关”和奖励数字。世界应该永久记录玩家做过的事情：

- 对应窗口永久亮起。
- 桌上留下玩家制作的饮品。
- 人物之后会再次出现，并使用或谈及这份饮品。
- 巷子逐渐增加植物、蒸汽、声音、招牌和来往的人。
- 某些后续事件会使用玩家先前制作的东西。
- 第一章结束时，所有变化共同组成一条重新活起来的巷子。

这种“我改变了这里”的视觉证据，是主要的自我成就来源。经验值、钻石和成就标签只能作为补充。

## 十六、由玩家写完整的配方书

配方书不应只是答案查询器，而应成为玩家亲手建立的理解记录。

- 初始只显示部分材料、工艺或味道线索。
- 玩家成功发现以后，配方才被完整记录。
- 配方记录玩家第一次发现时使用的路线。
- 找到替代路线后，页面生长出新的分支。
- 重要原料旁可以记录真实文化信息与游戏化简化说明。
- 玩家可以保存并命名自己的配方变体。

示例：

```text
甜酸奶
├─ 牛奶 + 冰糖碎 → 甜牛奶
│  └─ 甜牛奶 + 发酵
└─ 酸奶 + 冰糖碎
```

玩家收集的不再只是图标数量，而是一张由自己逐渐理解、验证和扩展的味道关系图。

## 十七、无损试错与回酿

放松感首先来自玩家不害怕操作。

- 错误合成不消耗关键材料。
- 两个不匹配的物品柔和弹开。
- 不使用刺耳警报、红色大叉或羞辱性文案。
- 某些偏离目标的组合可以产生有趣的旁支物品，而不是垃圾。
- 旁支物品可以通过“回酿”恢复成一种关键原料。
- 连续多次偏离目标后，环境才提供逐步增强的方向提示。
- 提示只说明方向，不直接给出完整答案。

这个系统应该鼓励玩家主动产生“这样组合会变成什么”的好奇，而不是担心材料是否会被浪费。

## 十八、短而有触感的制作动作

第一章不设置长时间发酵等待，但工艺也不能全部表现为两个图标碰撞。

可以加入持续约 2–5 秒的短操作：

- 轻轻旋转搅拌。
- 拉动滤布收紧。
- 控制倾倒速度。
- 把花材撒入酒酿。
- 在合适的时机停止加热。
- 左右晃动容器让材料融合。

这些动作的设计原则：

- 不设置严格失败判定。
- 操作差异主要影响视觉、声音、成品个性或味道倾向。
- 第一次出现时帮助玩家理解工艺。
- 玩家掌握后允许快速完成或跳过，避免后期成为重复劳动。
- 操作过程本身要有舒适的声音、阻尼和视觉变化。

它们让玩家感觉自己参与了制作，而不是点击按钮后等待进度条。

## 十九、让玩家选择下一份需要

游戏不应永远线性指定下一关。巷子里可以同时出现两到三处生活需要：

- 花店正在准备收摊。
- 晚归的人刚打开灯。
- 小店要为几位朋友准备一桌饮品。

玩家可以自己决定先处理哪一处。所有选择最终都能推进章节，不设置错过内容或选择错误的惩罚。

目标仍然清楚，但前进顺序由玩家决定。这能同时增加自由感、代入感和再次游玩的兴趣。

## 二十、自由酿造台与个人配方

第一章是学习宝珠的制作语言，自由酿造台是使用这种语言。

完成第一章后，玩家可以：

- 自由选择基底、工艺、花材和温度。
- 立即观察味道属性如何变化。
- 保存和命名自己的饮品。
- 把自创饮品送给某位住户。
- 观察不同人物对同一饮品的不同反应。
- 将满意的作品写入个人菜单。
- 在后续事件中再次制作或改良自己的作品。

长期目标因此从“通过所有关卡”转变为：

> 学会传统以后，创造一种真正属于自己的味道。

游戏最终可以让整条巷子的人聚在一起，由玩家使用一路理解的原料和工艺，制作一份没有标准答案的饮品。它成为玩家菜单中的代表作。

## 二十一、成就感不依赖评分

第一章不使用速度、连击和三星评价制造成就感。游戏记录玩家做过什么，而不是评价玩家比别人强多少。

适合的手艺记录包括：

- 第一次理解“厚与轻”。
- 找到一种新的花香路线。
- 用不同方法满足了同一种需要。
- 在没有提示的情况下完成一份复杂饮品。
- 创造第一份个人配方。
- 让一户窗口亮起。
- 让整条巷子重新有了生活。

这些记录可以进入配方书、个人菜单、巷子变化或回忆界面，不需要在完成瞬间连续弹出多个奖励窗口。

## 二十二、自由度的成长顺序

自由不能一开始全部开放，否则新玩家会失去目标。建议按以下顺序成长：

1. **第一章前半段**：唯一答案，快速理解基本操作。
2. **第一章后半段**：出现少量替代路线，开始自己推导。
3. **第二章**：人物提出味道属性，而非指定唯一成品。
4. **后续章节**：加入容器、温度、制作动作和更多路线选择。
5. **长期玩法**：自创饮品、个人菜单、自由赠予和改良配方。

这种顺序让目标感逐渐转化为自主感，而不是突然从教学进入完全无方向的沙盒。

## 二十三、不应加入的刺激系统

以下系统可能制造短期活跃和重复游玩，但会破坏宝珠游戏的核心体验：

- 每日连续登录和断签惩罚。
- 体力限制。
- 强制等待数小时。
- 限时订单。
- 制作速度排行榜。
- 随机稀有材料抽取。
- 配方成功率。
- 错误合成后材料永久消失。
- 为了三星评价反复重刷。
- 依赖连击、操作速度或朋友排名证明能力。

游戏可以有挑战，但挑战应来自观察、推导和表达，而不是焦虑、运气或重复劳动。

## 二十四、玩法原型优先级

在继续增加关卡前，优先验证以下玩法原型：

### 原型一：永久变化

完成饮品后，一户人家从暗到亮，并在之后持续保留变化。

验证目标：玩家能否自然理解自己的制作进入了具体生活，并愿意继续看看下一户会发生什么。

### 原型二：多个正确答案

同一个味道需要至少允许两种正确做法，并产生不同但都积极的反馈。

验证目标：玩家是否觉得自己在选择和表达，而不是猜系统唯一答案。

### 原型三：无损试错

允许玩家主动尝试未知组合，错误不会损失关键材料，旁支产物可以回酿。

验证目标：玩家是否愿意主动实验，并认为过程轻松而不是紧张。

### 原型四：个人配方

让玩家保存、命名并送出一份自创饮品。

验证目标：玩家是否会把作品视为“我的饮品”，并产生比领取普通奖励更强的成就感。

这四个原型分别验证使命、自由、放松和自我成就，也是后续玩法扩展的基础。
