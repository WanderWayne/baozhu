玩法与动作系统设计文档 V1.0
#0. 设计目标（Design Pillars）
核心目标
操作极简（拖、停、靠近）
体验逐步变复杂（动作叠加）
沉浸、神秘、有“被世界回应”的感觉
不靠反应、不靠数值、不靠说明文字

设计参考
Infinite Craft（自由合成）
纪念碑谷（少动作，多组合）
Zelda 神庙（机制逐关引入）
Journey / Inside（环境叙事）

#1. 核心玩法：合成（Craft）
1.1 基本描述
玩家将两个或多个“词语/物品”拖到一起
在满足条件时生成新物品

1.2 特点
非格子化（任意位置）
合成有延迟（不是瞬间）
合成结果可模糊展示（？？？ → 揭晓）

1.3 作用
游戏的“语言系统”
所有其他动作都围绕合成展开

#2. 基础动作系统（第一层：安全动作）
2.1 靠近（Approach）

是什么：
某些效果只有在物品靠近特定区域/物品时才会发生
玩家操作：
拖动物品靠近另一个物品或区域
系统判断
距离 < 阈值 → 激活状态
体验价值
空间开始有意义
不是“哪里都一样”
比如：只有两个物品在背景是水的地方才能合成

引入关卡

第一关
2.2 停留 / 静置（Pause）
是什么
玩家什么都不做，本身就是一种有效操作
玩家操作
将物品叠放后系统提示：“有时候，时间才是答案”，玩家不移动物品不移动
系统判断
停留时间 > X 秒 → 状态变化
示例
发酵
融合
显现隐藏属性
体验价值
强烈仪式感
纪念碑谷/Journey 的“慢体验”

第一关

#3. 核心扩展动作（第二层：逐关引入）
3.1 旋转 / 对齐（Rotate / Align）

是什么
物品需要处在正确朝向/状态才能生效
玩家操作
两指旋转
或拖拽到对齐位置
系统判断
角度 ∈ 合法区间
示例
桂花朝上才能释放香气
杯口对齐才能倒入
体验价值
仪式感
非技巧型思考
引入关卡
第二关（桂花 / 花类）

3.2 踩点 / 激活（Trigger）

是什么
将物品放在特定点位，改变环境状态
玩家操作
拖物品到“节点”上
系统判断
节点被占用 → 世界状态改变
示例
冷 / 热
开 / 关
某类合成启用
体验价值
世界“回应”玩家
引入因果关系

3.3 移动环境（Shift Space）

是什么
玩家不动物品，而是改变世界结构
玩家操作
拖动世界部件 / 滑动背景
系统判断
空间结构改变 → 可达性变化
示例
水位升降
昼夜切换
季节轮盘
体验价值
高级但直觉
强烈纪念碑谷感

#4. 高级动作（第三层：神秘感来源）
4.1 分离 / 提取（Extract）

是什么
从合成物中取出一部分
玩家操作
长按 → 拖出
系统判断
是否可被提取
示例
从酸奶碗取出坚果
从失败品提取“经验”
体验价值
可逆性
减轻试错焦虑

4.2 共鸣 / 自动吸引（Resonance）

是什么
某些物品自动靠拢或排斥
玩家操作
无直接操作，靠位置触发
系统判断
属性匹配 → 吸引 / 排斥
体验价值
世界像“活的”
减少操作负担

#5. 核心新增机制：互相排斥（Repulsion）

这是你刚刚提出的，非常重要的一种“空间难题机制”

5.1 机制定义

互相排斥：
某些物品之间存在排斥力，无法靠得太近。

5.2 玩家看到的表现（非数值）

靠近时轻微抖动
推开
光效变冷
发出低频音
⚠️ 不显示“无法合成”的文字

5.3 系统规则（示例）
A 与 B 排斥
B 必须靠近 C 才能激活
A 与 C 可以合成
但 C 激活状态下会排斥 A

5.4 典型谜题结构（你提到的那种）
目标：
合成 A + C
条件：
C 必须靠近 B 才能“激活”
B 会排斥 A
解法：
玩家需要：
先让 C 靠近 B 激活
保持 B 与 C 的距离刚好
在排斥范围边缘让 A 靠近 C
在“临界状态”完成合成

5.5 体验价值
不靠手速
不靠复杂规则
靠“感觉”和空间判断
非常接近 Zelda 神庙的物理谜题

5.6 引入时机
第四或第五关
玩家已经理解：
靠近
停留
环境状态
